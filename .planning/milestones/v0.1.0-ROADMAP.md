# Milestone v0.1.0: Initial Release

**Status:** ✅ SHIPPED 2026-03-21
**Phases:** 1-7

## Overview

This milestone turned `jqx` from a partially working prototype into a release-ready shared-core system. The shipped scope covered shared-core jq compatibility, canonical MoonBit and JS/TS public APIs, jq-compatible CLI workflows, standalone schema adapters, release dry-run workflows, MoonBit package readiness, cross-surface documentation, and complete Nyquist validation coverage for the full milestone.

## Milestone Summary

**Key Decisions:**

- Keep one shared MoonBit semantic core across CLI, JS/TS, and MoonBit rather than splitting behavior by surface.
- Center the MoonBit public package on standard `Json`, with JSON text reserved for the compatibility lane.
- Make the JS/TS root package a direct runtime first, and keep `/bind` as the explicit advanced integration lane.
- Publish adapters as standalone npm packages rather than root-package subpaths.
- Require auditable release dry-runs and complete Nyquist validation coverage before milestone closeout.

**Issues Resolved:**

- Closed the remaining jq compatibility, fidelity, and object-order proof gaps in the shared core.
- Removed internal-type leakage from the MoonBit public boundary.
- Aligned JS/TS runtime exports, compiled execution, and package-output proof.
- Closed CLI workflow parity for common jq options and direct-input behavior.
- Reconciled the remaining partial validation artifacts so the milestone audit no longer reports tech debt.

**Issues Deferred:**

- Routine `mooncakes.io` publishing remains a next-milestone operational concern rather than a shipped workflow.
- A dedicated `@shina1024/jqx/query` package surface remains deferred to a later milestone.
- Linux CI remains the authoritative full jq differential proof path while local Windows wrapper behavior stays secondary.

**Technical Debt Incurred:**

- No blocker-level product or validation debt remained at closeout, but post-`0.1.0` release operations and query-surface expansion are intentionally deferred.
